# Prüfliste für Bounded Contexts

- [ ] Der Kontext hat eine klare Verantwortlichkeit.
- [ ] Wichtige Begriffe sind lokal definiert.
- [ ] Die Eigentümerschaft ist ausdrücklich.
- [ ] Externe Konzepte werden übersetzt.
- [ ] Upstream- und Downstream-Rollen sind dokumentiert.
- [ ] Die Dateneigentümerschaft ist klar.
- [ ] Der Kontext kann sich unabhängig weiterentwickeln.
- [ ] Eine Netzwerkgrenze wird nicht automatisch angenommen.
