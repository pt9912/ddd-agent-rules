# Prüfliste für Pull Requests

Gib für jeden Befund an:

- Regel-ID
- Schweregrad
- Datei und Stelle
- Problem
- fachliche Konsequenz
- empfohlene Änderung
- erforderlicher Test

Lehne Änderungen ab, die verbindliche Regeln ohne ausdrückliche und vertretbare Ausnahme umgehen.

Halte eine reine Prüfaufgabe schreibgeschützt. Empfehle Änderungen und erforderliche Tests, verändere Dateien aber nur, wenn auch eine Implementierung verlangt wurde.
